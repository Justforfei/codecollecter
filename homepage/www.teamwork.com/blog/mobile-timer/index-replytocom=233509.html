Features & Updates

Mobile Timer

Posted May 30, 2014

It’s been a busy month for mobile here in Teamwork, with a raft of updates, features and bug fixes in the works. Top of the list: Timers for Projects and Tasks.

The Teamwork.com app always had the ability to log time. But you had to be a disciplined person to use it: first you’d have to remember to keep an eye on the clock and then you’d have to remember to go back to the appropriate screen in the appropriate project to actually log it. But when you’re busy, it’s easy to get carried away with just getting the work done. If you forgot to check the clock before you started, it’s time to start guessing.

With the new Timer, when you to go to a Project or Task, you’ll be able to start the timer there and then. A small bar will appear on every screen, so you can move freely around the app and the timer will always be visible. Tap on the timer bar and you get a big, beautiful Timer Screen which you can leave open: perfect to help keep you focused on the job at hand.

When you’re done, tap the upload button, add some extra information, or not, and then hit Save. Simple.

How do I get started?

On both the Project and Task screens, the Options menu now includes a Timer button. Tap on it to start the timer. On Task screens, there’s also a big “Timer” button in the time section. Just expand the time section as normal by tapping on the time header: there you’ll see the list of time already logged to the task and both a “Timer” and a “LogTime” button.

When will it be available?

For iPhone : it’s in the latest update in the App Store. For Android : it’s ready as well, but we want to make sure it works well on the varied devices out there. So we’re running a short beta test. Just let us know at beta@teamwork.com, especially if you have an unusual Android device, and we’ll send you the app. After a very short beta run, we’ll push it out to Google Play.

Some of the other notable features this month:

  • Support for “Task List Defaults” that were rolled out on the website last month
  • Login screen is now automatically shown if authentication fails. So no more of those terrible “403 error” alerts!
  • New Milestone Filters: ‘Upcoming’ and ‘Late’
  • Milestone Viewer : show (multiple) attached task lists – tap on ’em to jump to ’em
  • Milestone Viewer : show project – also tappable
  • On the quick add screen, we’re trying out a couple of new gestures; if all goes well, we’ll roll them out to the rest of the app:
    • Swipe Left/Right on assign-to rows to clear or set to ‘me’
    • Swipe Left/Right on due dates etc. to decrement/increment the date

And for a more complete list:

  • All Projects : companies sorted alphabetically
  • Better support for custom domain names and long passwords
  • Much friendlier time stamps on comments
  • Can now change company when editing a project
  • On bigger devices, there’s now a bigger text area on all of the creator screens ( Task Creator, Milestone Creator etc.)
  • Files List is now paged for extra speediness

And a nice bunch of fixes:

  • Sorting Milestones was showing duplicate milestones
  • Notify from Create Task
  • Task priority is no longer automatically set when creating a new task
  • Couldn’t press return in the description field
  • Billable switch was ignored in some particular cases
  • Search fails with Russian characters
  • Events : edit : choose who to notify : showed error alert
  • Increased Viewer font sizes

An end to Heuston

As you might know, we’re a proud Irish company. So when it came to naming the alerts, we decided to give the normally dry warnings a slightly playful twist with “Heuston we have a problem”. What you might not know is that Heuston is the largest train station in Ireland. However, too many people thought it was a simple spelling mistake, so as of version 1.0.8, Heuston becomes Houston.

We’ve just hidden a different bit of Ireland somewhere else in the app instead
