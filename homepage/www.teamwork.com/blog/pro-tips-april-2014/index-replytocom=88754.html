Teamwork Projects Pro Tips – April 2014

Posted April 22, 2014

Following on from Teamwork Projects’s March’s Pro Tips, this month we again focus on reporting. But this time we’re looking at reporting from across several projects.

1) Projects – Generate activity report

The activity report is a great option to view all the activity that has happened on all your projects, whether for everyone involved or for individuals. You can break the activity report down further by time period and by the items you want to view.

 

You can find the activity report under the Projects tab in the upper left of the screen, and then under the Options menu in the left column. Select the report options from those given in the pop-up pane and then hit the “Generate Report” button.

 

2) Projects – Project Chart

Another really useful option is the Project Chart, also located under the Projects tab and directly under the “Your Projects” heading. The Project Chart displays all your projects in a Gantt-style view.

To use the Project Chart, all your projects must have the project start and end dates entered. The chart shows the timelines for your projects based on the milestones you’ve entered.

 

3) Everything

 The Everything section, also accessed by tab at the upper left of the screen, gives you a choice of options to report on all items, including:

  • All active tasks – Choose the sort options you need for the active tasks; export options are on the far right of the screen and are PDF, Excel or an All Tasks Report that runs within the system.

  • All completed tasks – Reports on all completed tasks for a time period, either for all people or just those assigned to an individual; export to PDF or to Excel.

  • All milestones – Choose the report format from the Options menu at the bottom left of the screen; generate the report to PDF, Excel or export to an external calendar via iCal.

  • All time – This is a report of time logged across all projects; the report generates within the system then gives options to export to PDF, Excel or CSV.

  • Workload – Based on the estimated time assigned to each task; reports how much time is allocated to either a person or a project over a specified time period; export option is to PDF.

4) Calendar – Print Agenda

At some point, you’ll probably need to see all upcoming events together in one place. At the moment, the calendar view in Teamwork is limited to one month (but we’re thinking about changing this.)

To get a report of all events:

  • Navigate to the calendar

  • Select the options in the upper right hand section

  • Choose Print Agenda

5) People – Login Details

You can find the People tab also at the top left of the screen. The Login History report gives a quick snapshot of people’s activities on the system, including when they last logged in, how many times they’ve accessed it and much more. Only administrators in the owner company can access this report. Export options for this are to PDF, Excel or CSV

 

6) Settings – Export option

The backup that’s generated through the Export option is a MySQL data file that you can run against a database on your local MySQL server. You can download MySQL fromwww.mysql.com. Once you run your local database server and run the .sql file against it, all tables and data will be restored so you can run your own queries against it.

Internally we use www.navicat.com which lets people extract data in about 20 different formats, such as Excel/XML etc, as they need, and has visual tools for joining tables and running reports.

There are loads of free apps out there for manipulating and reporting from MySQL such as MySQL Workbench (http://www.mysql.com/downloads/workbench/)

The Export option is available so you can obtain a copy of your data at any time if, perhaps, you have concerns about the service or are bound by policies to always have the option of retaining your data. Also, any information added to Teamwork Projects is the property of the person who owns the Teamwork Projects account, so we believe it’s only fair to give you access to your data at anytime. Other people use the backup as a fast and flexible way of running additional reports that we don’t have without having to use the API (developer.teamworkpm.net).

Unfortunately due to the nature of the Teamwork Projects platform, the architecture, backup and security that are in place, we can’t accept a MySQL file back into Teamwork Projects, so the export is one way only. If you need to remotely manage your data outside the Teamwork Projects App and UI, then the API is the best way to go.

As always we hope you find this helpful. Please keep the feedback and suggestions coming.
