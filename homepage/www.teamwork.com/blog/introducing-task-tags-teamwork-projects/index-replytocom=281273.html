Introducing: Task Tags for Teamwork Projects

Posted May 29, 2015

As a company, here at Teamwork.com only one thing ever matters to us, delivering for our customers.  That’s why we develop based on what you, our customers, ask for. We want to make sure that we’re building features that will improve your daily experience of using Teamwork Projects.

Introducing your most requested feature ever, Task Tags for Teamwork Projects. Now you have whole new way to keep track of your tasks with the ability to create all kinds of tags and labels for them and make them searchable and easy to report on. It’s a simple addition that creates incredible options for you in terms of planning and reporting.

Getting Started

It’s easy to get started, you can create, and add tags really simply.

 

Reporting on Tags

You can run reports to single out activity around a particular tag, or tags.

Searching Tags

Searching tags couldn’t be easier.

Replacing Tags

Replace tags as things about a particular task change.

Merging Tags

We’d made it simple to merge different tags into one, to help.

Changing Colors on Tags

You can even change the colors on a tag to make it visually easier to find out infomation at a glance.

Task Tags is first of a whole series of big improvements that we’re making to Teamwork Projects this year. We’ll be rolling out updates to add task functionality to the mobile applications, iOS and Android in the next couple of days. We’ll also be adding further task functionality.

As always your feedback is vital so get it touch via email at support@teamwork.com, Twitter, or Facebook.

Just to add to what others already have said. As part of a team, sometimes tasks are assigned to somebody and when the work is done, somebody has to check – usually, the one who initially assigned the task.

In this case, a task status of “open” or “complete” won’t do – what I do (with other project management systems as well) is to create a “finished” tag and apply it to the task. This way, whoever assigned the task will be able to know that this is finished, check it and close it.

Usually tags that I use are “In Progress”, “Finished”, “Won’t Implement”, “Pending Feedback” (in case someone has assigned a task to you but you need more information), and “Won’t Implement”. Not with these exact literals, but I think you can understand the concept.

Joshua Smith June 1st, 2015

Here are some common use cases:

1. Finding things. You can use it as a flag to make it easier to find items later. Kind of like you do with e-mails.
2. Using it as a status indicator: You can create tags such as Started, In-progress, Waiting for Approval, etc..
3. Use it to create ad-hoc workflow. This would be similar to #2. This would be good if they allowed you to create custom tags for each project or for a task list. But being that the tags are global you are limited in how effect it can be used for workflows.
4. Currently you group tasks into task lists. You also could use tags to create new way to create groups. For example you have a list of tasks for a project. To complete some of the tasks needs to purchase items. Instead of creating a purchase list, you could just add tags such as purchase approval, purchase ordered, purchase received, etc… Or do one tag purchase and then use some status for tracking. That way the person doing purchasing can quickly see any tasks they need to do without you having to specifically assign them to a task, etc..

Michael Rollins May 30th, 2015

Let’s say you’ve got a workflow whereby you make a code or content change for a client, test it internally, send it to the client for confirmation, get confirmation/approval, then roll out the change. With tags you can add a “status” to a task to reflect what stage in the workflow it is in. Without tags you might have to group them into task lists, or prepend something descriptive in the task name.

You might also want to be able to mark tasks as being on hold, or needing review, etc. Finally, tags make it possible to create lists or groups of tasks across clients or projects based on any criteria you like, such as “In Trouble” or “Watch Closely” or “Late”.

Make sense now? It’s a really flexible and powerful feature. Thanks, Teamwork!

Joshua Smith June 1st, 2015

What they need is what other sites do, they create custom workflows. A custom workflow is where you designate a series of tags with a scope, the scope being a workflow. Then you assign workflows to a tasklist or project. That way you don’t have to clutter tags up for doing workflows. More complex workflows not only define the tags in a group but the relationship between the tags. Such as planned->in-progress->initial complete-> manager review -> accepted & completed. With you only able to select the next tag in the sequence. Others even allow loops. (see dobambam.com for an example of this)
