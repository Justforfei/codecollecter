One of Teamwork.com’s most popular extras, the Desktop Timer has gotten an upgrade. 

Posted June 30, 2014

One of our most popular tools, the Teamwork Timer has always been a big hit with our users that demand accurate time tracking on projects, for record keeping or billing purposes. Building on feedback from users, we’ve updated the timers, last month was the turn of the mobile timers and now this month, we’re delivering the brand new desktop timers app.

  • We built our timer app with everyday use in mind; you can have it open all the time in full view or switch to a more discreet compact view.
  • You can easily import all your upcoming tasks from Teamwork as timers and work on what’s due next.
  • If you step away from your desk and forgot to pause the timer, don’t worry! You can have the app automatically pause your timer when the computer is idle and resume it when you get back to working.
  • Did you log your last time log as billable? We’ll remember that next time! Did you choose to reset the timer but keep it open so you can log more time later? We’ll remember that too…

The teamwork.com desktop timer works on any desktop – Mac, Windows and Linux and now auto-updates so you stay up to date as we add new features so you’ll always have the latest version of the software. You check out all the details right here.

As always we appreciate any feedback on the update.

Nicholas Istre July 26th, 2014

Just a heads up for anybody running a recent verion *ubuntu distro (I’m running Kubuntu 14.04), the app as downloaded will return the error:
$ Teamwork\ Timer
Teamwork Timer: error while loading shared libraries: libudev.so.0: cannot open shared object file: No such file or directory

A workaround is to create a softlink to the existing libudev.so.1.3.5 library (Again, at least on my Kubuntu 14.04 setup, other distros may have a different version):
$ sudo ln -s /lib/x86_64-linux-gnu/libudev.so.1.3.5 /usr/lib/libudev.so.0

This will be needed until the webkit library used links against libudev.so.1 instead.

Hello Michal,

Inside the archive you’ll find the “Teamwork Timer” executable. Depending on your Linux distribution, you may need to give it permission to execute.

If you are using Ubuntu, you should be able to right-click, go to Properties > Permissions tab and check the box “Allow executing file as program”. Otherwise, this can be done via command-line: chmod +x ./Teamwork\ Timer.
You should then be able to double-click the timer app to run it.
