December 2013 Updates and Fixes

Posted December 23, 2013

Well, we have finally come to the end of 2013. It’s been an amazing year with a huge amount of updates and new features released here at TeamworkPM. Thanks to you all for all your feedback and support throughout the year.

Here is the list of some of the smaller updates and fixes that were rolled out in December. We hope you are all looking forward to New Year and, for those of you who are interested, the full list is available to view straight from our SVN Repository on our website here.

Updates and Features

  • Disabled activity log when importing files

  • Update Tasks/Comments: Standardised order or Task/List/Project links

  • KO View – Comments file attachments

  • more HTML markup (in progress)

  • modified etag for available people call

  • moved usePushState from API to KO Comment Model

  • People removed alt email addresses from expanded listings view

  • Comments Edit/Delete comments permissions (in progress)

  • User page verified and tested left pane User options (better KO binding event functions syntax)

  • fix removed button downgrade/upgrade from current user edit form (only available now in ‘User options’ left pane)

  • Fix Everything: All Milestones: Listing not showing company project is associated with

  • Comments objectType/objectId options for KO Comment view (task”, “milestone”, “file”, “notebook”, “link)

  • MVVMAdding/Editing tasks – Tweaks to keep task editors working when new mode flag off

  • MVVMAdding/Editing tasks – think i’m there apart from bus testing

  • UI improvement: tipped and quick link menu for user, company, project and task name (checked permissions)

  • MVVMAdding/Editing tasks

  • Dynamic loading of project permissions

  • Translations in new add/edit template

  • Update Tasks: Reminder icon shown in hidden options when no reminders are set

  • UI+KO Data Model: comment date formatting

  • MVVMAdding/Editing tasks – big update, getting there, big optimizations by cancelling event bubble

  • JSNew system for remembering per-project permissions in javascript userInfo struct.

  • UI Project people page: new visual effect when user is added/removed from a project (highlight/opacity)

  • UI: Item name and related object link (use ql/push state option)

  • Box.com sorting

  • Third party file storage Decoupled the client-side sorting and added it to Google Drive

  • Integrations Box.com: clearer text when no directories found inside root

  • Update Box.com: Switched completely from ‘Box API V1’ to ‘Box API V2

  • Dropbox link to root folder when user browses to folder they don’t have access to, which isn’t root

  • Work queue system uses error callback to handle request timeouts

  • E8 excanvas.js loaded in d.cfm before tipped libraries. Markup for project tipped menu now valid

  • TWDATA + objectType, objectId, renamed variables for consistency with API

  • General Replaced most confirm()s with confirmation lightboxes

  • Archiving projects – allow the option to remain on the projects tab. Currently redirects to the archived projects.

  • koSwitching adding and editing of tasks to knockout

  • Header Removed bold property for element with proxima nova as a font

  • User page quick launch upload photo form clicking on avatar

  • DO import added linking of files to comments/tasks

  • Dropbox browse: files and folders separated like Windows

  • Dropbox Browse: client-side sorting

  • Google Drive attachments now show appropriate icons.

  • Changed getTasks to getAllTasks where needed.

  • Listing Knockout Templates improvements: remove all options passed to view, use of Knockout Templates Loader for faster instantiation

  • Sub-tasks are now hidden by default

  • completed sub-tasks text change

  • AllTasks() returns struct instead of query – will allow optimization of countOnly=true case

Bugs

  • Fix for importing of files to archived projects

  • Bug Fix Send notification to all appservers when setting changed in General to invalidate account cache

  • Bug fix IE9/8: Notebooks: If content is added don’t prompt to leave page on Category/Notify/Privacy menus

  • Milestones Fixed week numbers broken on firefox

  • Bug Fix Roles: Add/Edit: Focus on long pages causing lightbox to jump out of view

  • Flies Fixed move files between projects

  • Minor bugMVVm Task Editor:Private Lists: When a task is being added to a private list we need to disable the Privacy tab on the Add/Edit form

  • Minor bug fixTask editor in MVVM:Selecting ‘multiple people’ does not pop-up multiple choice

  • File drop Fix for when files are turned off on a project

  • Milestones Fixed content loading twice due to old pushState code (milestone tabs behave differently now)

  • fix Dashboard: link to user detail page should not be available unless user have permissions

  • Fix side bar resize in chrome

  • Fix Task list templates: Don’t filter by user if ‘user filter’ is set in a normal project

  • Third party file browsers tiny fix

  • Fix People listing – “Expanded” view has no edit link+ refactoring variables naming + better permissions check

  • fix User details – Projects number can overlap text

  • Notebook: buttons overlap title

  • bug fixCalendar – Edit “all day” event, when “All Day” is unchecked, time options do not show

  • fix Add people to project, on select dropdown, hit enter now select company

API Updates

  • API extended file attachments data

  • API/KO Model lockdownId

  • Update API: Time Entries: Added “todo-list-id” in to response

  • KO Model + itemName, commentLink, usePushState (consistent API/TWDATA)

  • API/KO Model + companyName, projectName, type, nr. notified people, privacy

  • API + property: isRead

  • KO Comment data Model matching API (first draft – in progress)

  • API + APICall_GET_comments

  • API Message and reply attachments now include external file extra data

  • APIAll old “todo_lists”/”todo_items” calls now can use “tasklists”/”tasks” – old calls will be deprecated sometime
