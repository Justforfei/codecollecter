January Updates

Posted January 29, 2015

Even with the launch of Teamwork Chat, we’ve worked equally hard to continue to roll out updates and fixes to Teamwork.com. Based on your comments and feedback we’ve worked hard to make some small adjustments that had cropped up:

Comments:

  • Show the authors company name next to the authors name now.

Forms:

  • We now support submitting tasks/milestones/comments/messages etc using ctrl+enter/cmd+enter

Quicksearch:

  • Added an easy way to clear search term

Task Reminders:

  • Added ability to add a note to task reminders

Tasks:

  • Added option to parent task to bulk edit sub-tasks

Notebooks:

  • Improved pasting images in to notebooks
  • Improved pasting content from Google Docs

Tasks:

  • Quick entry: Added ~ option to quickly set estimated time
  • All Time: Added bulk update option to mark time logs billable or non-billable

Project Chart:

  • Added ability to set a custom start and end date to render the chart

Tasklists:

  • New task lists are added at the bottom now to make consistent with how new tasks are added

API:

  • Added ability to only return calendar events from a certain type
  • Added ability to mark a comment as read

As always if we can help out in any way, fix something you’ve spotted or you just want to say ‘hi’ drop us a line at support@teamwork.com

Hi Guys,

Just thought I would congratulate you on such a great product. I’ve been testing out project management systems for the past 2 months and have demo’d over 30 different systems – eventually coming back to TW. I’ve been looking for a PM mostly for managing our website/backend system development, that would provide a connected hub between designers, developers, directors, and department managers across sales, product and marketing that all need their input on the systems development and roll out.

While no system did everything we want, TW is the one with the best overall features, usability and proactive roadmap – well done!

Now my suggestions as to how I think it could be improved:

1/. Provide a dedicated suggestions area, so people can comment on ideas and provide some voting structure ( I looked for one before positing this but could not find one).

2/. Improve you chrome extension so that I can paste in a screenshot and add attachments. Most of our tasks require a visual aid – a lot of annotated web screen grabs. Take a look at http://www.yodiz.com/ for a well implemented example.

3/. Different stages for tasks. At the moment there are only 2 stages Done/Not Done. The way we work (and I am sure others) is that a task runs through stages like Not Started, Started, Ready For Verification, Completed. The ready for verification is important as it allows the task initiator to find all the tasks they need to check have been done satisfactorily before completion. In effect this is adding some simple workflow, which could be done without compromising usability. I would imagine this could be implemented by a drop down hover state over the completed checkbox giving other options. If this were a setting you could turn on/off you would not effect people who don’t want it – a lot of other systems I have used in the past allow you to configure your own stage names, so its not one size fits all. I think this small addition would be make a very big difference,.

4/. Tags – I believe this is on your radar after checking the roadmap.

5/. Card View / Kanban – This I think could be a killer function, that would all the customers to work the way they want to work. I have IT dev’s wanting card boards for agile, while Directors want to see the Gantt, and others wanting everything in-between or in combination. If you did my point 3 with regards to stages, then you could have the board columns as each stage. This in my view would be a killer addition – there are so many blogs posting about their PM tools which have to use a Card style PM in conjunction with another PM it seems there is a huge opportunity if you can make them in one tools – Read this (great review of PM) as an example: http://charlesngo.com/teamtools/

Anyway, those are my thoughts – would love to here the thoughts of others and if any of the above would be considered.

Keep up the good work.

Daniel Mackey February 1st, 2015

Hi James,

We already do signature matching based on familiar signing off words if in the lower percentile of the email and matching on a persons full name. We have quite a set of rules and matches but we err on the side of caution so unfortunately some signatures still make it through. We’re constantly updating the rules so if you have one you’d like us to look at just drop us an email.

For images it’s more difficult because we don’t actually always know if an attachment was inline or attached. We match based on Hashes of previous images from the same user under a certain size threshhold with a match value of 2. This means we’ll always let the first 2 through but the 3rd and subsequent should be ignored if the hashes match. The #End directive will have no bearing on attachments but any text after this will be completely discarded.

Hope this helps,
Dan.
